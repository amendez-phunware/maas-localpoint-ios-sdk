ILPMessageProvider Protocol Reference

Conforms to NSObject
Declared in ILPMessageProvider.h

Overview

A message provider acts as a store for messages deemed "relevant" by the SDK.

@warnning A message provider can be used by an application in two ways:

  1. synchronous - The provider can be queried for messages currently known to it.
  2. asynchronous - An application can register a {@link [ILPMessageListener](../Protocols/ILPMessageListener.html) listener} that will be notified when messages known are added to, deleted from or modified in the provider.

@version 1.1.0

Returns all messages filtered by the given filter and sorted by the given ordering.

Parameters

The filter to be applied to the messages to be returned (if nil, acts as a filter that accepts all messages; see {@link ILPMessageFilterFactory#getAllFilter()})
